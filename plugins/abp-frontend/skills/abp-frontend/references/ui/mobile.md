# UI — Mobile (React Native + MAUI)

> Reference for ABP's two first-party mobile UI options — the open-source React Native (Expo) startup template and the commercial .NET MAUI startup template — covering when each is appropriate, how each consumes ABP backends via OpenIddict, and the configuration steps unique to mobile clients.

## When to load this reference

- User asks 'how do I add a mobile app to my ABP solution' or 'what mobile options does ABP support'.
- User is choosing between React Native and .NET MAUI for an ABP project.
- User runs `abp new ... -m react-native` or `abp new ... -t maui --old` and needs setup help.
- User reports the React Native app cannot reach the backend at localhost from an emulator/device.
- User asks how to configure OpenIddict so an Android emulator or iOS Simulator can complete the auth flow against a dev backend.
- User asks where in the React Native template to add a screen, reducer, or saga.
- User asks how to call ABP application services from MAUI (C# Client Proxies, RemoteServices appsettings).
- User asks how to wire up SecureStorage for access/refresh tokens in the MAUI template.
- User asks how to do form validation in MAUI ABP (ValidatableObject<T>).
- User asks why their MAUI module needs AbpMauiClientModule and which DependsOn entries are required.
- User asks about multi-tenancy on a mobile client (tenant switcher, tenant resolver) in either RN or MAUI.
- User asks about supported MAUI target platforms (Android, iOS, Mac Catalyst, Windows) under ABP.

**Audience:** ABP developers who need to deliver a mobile client against an ABP backend — both teams evaluating which mobile stack to pick and developers actively scaffolding/extending the React Native or MAUI startup templates.

## Key concepts

- React Native startup template — Expo-based JavaScript/TypeScript mobile app shipped open-source with ABP. Generated by `abp new ... -m react-native` (or 'Mobile: React Native' in ABP Studio). Sits alongside the .NET backend and Angular/MVC frontend in the same solution. Reach for it when you want a free, cross-platform (Android + iOS + web) mobile client and your team is comfortable in JS/TS.
- MAUI startup template — Commercial (Team license or higher) cross-platform .NET client targeting Android, iOS, Mac Catalyst, and Windows. Generated by `abp new Acme.MyApp -t maui --old`. Reach for it when the team is C#-only, when you want to share code with a Blazor/server stack, or when you need Mac Catalyst/Windows desktop support that React Native doesn't offer out of the box.
- AbpMauiClientModule — `Volo.Abp.Maui.Client` package; the ABP module the MAUI template's `MyAppMauiModule : AbpModule` declares as its primary [DependsOn] entry. Pulls in MAUI-specific service registrations for Static HTTP client proxies, language management, settings, etc. [uncertain — exact namespace path inside the package not directly visible in the 10.3 doc page, inferred from the NuGet listing and ABP support thread].
- Volo.Abp.Http.Client.IdentityModel / AbpHttpClientIdentityModelModule — Module the MAUI template depends on so that the OpenIddict/IdentityModel-based password or authorization-code flow is available, and so that the Static C# Client Proxies can transparently attach access tokens to outgoing requests.
- AbpAutofacModule — The MAUI template uses Autofac as its DI container by default (matching the rest of ABP). Removing it requires hand-rewiring registrations and is generally discouraged.
- C# Client Proxies — In MAUI, application services exposed by the backend are consumed through generated/static client proxies (configured via `RemoteServices` in `appsettings.json`, with `Authority` and `BaseUrl` set to the running backend). Calls become regular `IXxxAppService` method calls instead of hand-rolled HttpClient code.
- OidcClientOptions — Microsoft.IdentityModel.OidcClient configuration object the MAUI template uses to drive the WebAuthenticator-based OpenIddict flow. Fields populated from appsettings: `Authority`, `ClientId` (e.g., `MyApp_Maui`), `RedirectUri` (custom scheme like `myapp://`), `Scope` (`openid email profile role MyApp offline_access`), `ClientSecret` if confidential client.
- WebAuthenticatorBrowser / Microsoft.Maui.Authentication.WebAuthenticator — The system browser strategy used to redirect to the ABP login page and back to the MAUI app via a custom URL scheme. Registered via `builder.Services.AddTransient<WebAuthenticatorBrowser>()` in `MauiProgram.CreateMauiApp`.
- AccessTokenHttpMessageHandler — Singleton DelegatingHandler that injects the stored access token into every outgoing HttpClient request and triggers refresh-token flow on 401. Registered with `builder.Services.AddSingleton<AccessTokenHttpMessageHandler>()`.
- SecureStorage — Microsoft.Maui.Storage.SecureStorage; the recommended store for access/refresh tokens in the MAUI template (uses Keychain on iOS, KeyStore on Android, DPAPI on Windows). Token keys are exposed via `OidcConsts.AccessTokenKeyName` and `OidcConsts.RefreshTokenKeyName`.
- ValidatableObject<T> — MAUI MVVM wrapper type used for view-model properties needing validation. Each instance has a `Validations` collection of rules and a `Validate()` method invoked from the view model. Used for the Login, Register, and Settings forms in the template. [uncertain — exact namespace not enumerated in ABP 10.3 docs, the ABP page directs readers to Microsoft's MVVM validation guidance].
- Built-in MAUI pages: Home (welcome/dashboard), Users (CRUD on application users with search), Tenants (multi-tenant management), Settings (language switcher, profile picture, password change, theme).
- Expo — JavaScript/TypeScript framework wrapping React Native used by the ABP template. Provides `yarn start`, `yarn android`, `yarn ios`, `yarn web`. The QR-code / Expo Go flow lets a physical device connect to the local dev server.
- Environment.ts — TypeScript config file in the `react-native/` folder. Holds `apiUrl` (backend API root), `oAuthConfig` (`issuer`, `clientId`, `clientSecret`, `scope`), and `localization` (`defaultResourceName`). For microservice backends, an `apis` map can be added with per-module `baseUrl` and `rootNamespace`. `localhost` MUST be replaced with the host machine's LAN IP for emulator/device testing.
- NativeBase — React Native UI component library used by the template (mobile-first, accessible). Wraps the app in `<NativeBaseProvider>`.
- React Navigation — Navigation library; screens are placed in `src/screens/` and grouped into navigators in `src/navigators/`.
- Redux + Redux Toolkit + Redux-Saga — State container. Actions, reducers, sagas, and selectors live under `src/store/`. Sagas handle async work (auth, API calls).
- Axios + interceptors — HTTP client with request/response interceptors in `src/interceptors/` for attaching the access token and handling 401 → refresh.
- Higher-order folders — `src/components/` (shared widgets), `src/contexts/` (React contexts), `src/hocs/` (HOCs), `src/hooks/` (custom hooks), `src/utils/` (pure helpers).
- OpenIddict (server side) — Both templates assume the backend uses OpenIddict (default since ABP 6.0). For both templates the `DisableTransportSecurityRequirement()` call in the HttpApi.Host module's `PreConfigureServices` is what makes plain HTTP work in DEBUG so emulators/simulators don't have to trust the .NET dev cert.
- Multi-tenancy — Both templates expose a tenant resolution mechanism on the login screen (tenant switcher / tenant text box) so the same binary can talk to multiple tenants. Server-side multi-tenancy config is unchanged from any other ABP UI.

## Configuration pattern

REACT NATIVE — There is no AbpModule on the JS side. Configuration is split across two files. (1) Backend `MyAppHttpApiHostModule.PreConfigureServices` MUST disable OpenIddict transport security in DEBUG so the emulator/device can complete the auth flow over plain HTTP:

  public override void PreConfigureServices(ServiceConfigurationContext context)
  {
      PreConfigure<OpenIddictBuilder>(builder =>
      {
          builder.AddServer(options =>
          {
              #if DEBUG
              options.UseAspNetCore().DisableTransportSecurityRequirement();
              #endif
          });
      });
  }

Kestrel must also bind `0.0.0.0` instead of `localhost` so non-loopback clients (Android emulator on 10.0.2.2, physical phones) can connect. (2) `react-native/Environment.ts` holds the client-side config:

  const ENVIRONMENT = {
    default: {
      apiUrl: 'http://192.168.1.10:44300',     // host LAN IP, NOT localhost
      oAuthConfig: {
        issuer: 'http://192.168.1.10:44300',
        clientId: 'MyApp_App',
        clientSecret: '1q2w3e*',
        scope: 'offline_access MyApp',
      },
      localization: { defaultResourceName: 'MyApp' },
    },
  };
  export const Environment = ENVIRONMENT.default;

MAUI — Configuration is the standard ABP module pattern plus `MauiProgram`. The module class wires up framework dependencies; `MauiProgram.CreateMauiApp` wires up MAUI-specific services and triggers `InitializeAsync`:

  [DependsOn(
      typeof(AbpMauiClientModule),
      typeof(AbpHttpClientIdentityModelModule),
      typeof(MyAppHttpApiClientModule),   // generated static proxies
      typeof(AbpAutofacModule)
  )]
  public class MyAppMauiModule : AbpModule
  {
      public override void ConfigureServices(ServiceConfigurationContext context)
      {
          var configuration = context.Services.GetConfiguration();
          Configure<AbpRemoteServiceOptions>(options =>
          {
              options.RemoteServices.Default = new RemoteServiceConfiguration(
                  configuration["RemoteServices:Default:BaseUrl"]!);
          });
      }
  }

`MauiProgram.CreateMauiApp`:

  builder.Services.AddTransient<MainPage>();
  builder.Services.AddTransient<WebAuthenticatorBrowser>();
  builder.Services.AddSingleton<AccessTokenHttpMessageHandler>();
  builder.Services.AddApplication<MyAppMauiModule>();
  var app = builder.Build();
  app.Services.GetRequiredService<IAbpApplicationWithExternalServiceProvider>()
     .Initialize(app.Services);

`appsettings.json` shipped inside the MAUI project supplies the OpenIddict + remote-service URLs and MUST point at the running backend (`Authority` and `BaseUrl` aligned). For tiered/microservice deployments, both the auth server port and every remote-service port must be reachable; on Android, run `adb reverse tcp:44305 tcp:44305` (and the auth-server port) from Visual Studio's adb prompt to forward the host port into the emulator. iOS Simulator already shares the host's network so localhost works directly.

## Code examples

### Disable OpenIddict transport security for mobile dev (backend)

_Required for both React Native and MAUI when running the backend over HTTP for emulator/device testing — without this, the auth flow fails with 'transport_security' errors._

```csharp
// MyAppHttpApiHostModule.cs
public override void PreConfigureServices(ServiceConfigurationContext context)
{
    PreConfigure<OpenIddictBuilder>(builder =>
    {
        builder.AddServer(options =>
        {
#if DEBUG
            options.UseAspNetCore().DisableTransportSecurityRequirement();
#endif
        });
    });
}
```

**Key lines:** DisableTransportSecurityRequirement() — Allows OpenIddict to issue tokens over plain HTTP. Wrap in #if DEBUG so production still demands HTTPS. Must run in PreConfigureServices because OpenIddictBuilder is a pre-configurable component.

### React Native Environment.ts pointing at LAN IP

_Switching the React Native template from `localhost` to a routable host IP so an Android emulator (10.0.2.2 maps to host) or a physical iPhone on the same Wi-Fi can reach the backend._

```csharp
// react-native/Environment.ts
const ENVIRONMENT = {
  default: {
    apiUrl: 'http://192.168.1.10:44300',
    oAuthConfig: {
      issuer: 'http://192.168.1.10:44300',
      clientId: 'MyApp_App',
      clientSecret: '1q2w3e*',
      scope: 'offline_access MyApp',
    },
    localization: { defaultResourceName: 'MyApp' },
  },
};
export const Environment = ENVIRONMENT.default;
```

**Key lines:** apiUrl + issuer must use the same routable IP. Android emulator's 10.0.2.2 alias also works in place of the IP. The `clientId` value matches the OpenIddict client seeded by the backend's `OpenIddictDataSeedContributor`.

### MAUI module declaration with required ABP dependencies

_Wiring up the MAUI app's root AbpModule so static client proxies, IdentityModel-based token handling, and Autofac DI are all available._

```json
[DependsOn(
    typeof(AbpMauiClientModule),
    typeof(AbpHttpClientIdentityModelModule),
    typeof(MyAppHttpApiClientModule),
    typeof(AbpAutofacModule)
)]
public class MyAppMauiModule : AbpModule
{
    public override void ConfigureServices(ServiceConfigurationContext context)
    {
        var configuration = context.Services.GetConfiguration();
        Configure<AbpRemoteServiceOptions>(options =>
        {
            options.RemoteServices.Default = new RemoteServiceConfiguration(
                configuration["RemoteServices:Default:BaseUrl"]!);
        });
    }
}
```

**Key lines:** AbpMauiClientModule — pulls MAUI-specific client services. AbpHttpClientIdentityModelModule — adds the OpenIddict-aware HttpClient handlers. MyAppHttpApiClientModule — generated static proxies for the backend's application services. AbpAutofacModule — keeps the standard ABP DI container.

### MauiProgram.CreateMauiApp DI + Abp Initialize

_Bootstrapping ABP inside the MAUI app builder and registering the OIDC browser + token handler._

```csharp
var builder = MauiApp.CreateBuilder();
builder.UseMauiApp<App>();

builder.Configuration.AddJsonStream(
    Assembly.GetExecutingAssembly().GetManifestResourceStream("MyApp.appsettings.json")!);

builder.Services.AddTransient<MainPage>();
builder.Services.AddTransient<WebAuthenticatorBrowser>();
builder.Services.AddSingleton<AccessTokenHttpMessageHandler>();

builder.Services.AddApplication<MyAppMauiModule>();

var app = builder.Build();
app.Services.GetRequiredService<IAbpApplicationWithExternalServiceProvider>()
   .Initialize(app.Services);
return app;
```

**Key lines:** AddApplication<MyAppMauiModule>() registers the ABP module graph against MAUI's IServiceCollection. The IAbpApplicationWithExternalServiceProvider.Initialize call is required because MAUI owns the ServiceProvider — ABP cannot build its own. WebAuthenticatorBrowser must be transient so each login uses a fresh browser session.

### MAUI appsettings.json with OpenIddict + RemoteServices

_Single source of truth for both the OIDC authority and the API base URL the static client proxies will hit._

```json
{
  "AuthServer": {
    "Authority": "http://192.168.1.10:44305",
    "ClientId": "MyApp_Maui",
    "ClientSecret": "1q2w3e*",
    "Scope": "openid email profile role MyApp offline_access",
    "RedirectUri": "myapp://"
  },
  "RemoteServices": {
    "Default": { "BaseUrl": "http://192.168.1.10:44300" }
  }
}
```

**Key lines:** Authority must match where the auth server is running (often a different port than the API in tiered/microservice setups — both must be reachable from the device). RedirectUri uses a custom URL scheme registered in Android's `AndroidManifest.xml` and iOS's `Info.plist` so the system browser can deep-link back into the app.

### Android adb reverse for tiered/microservice MAUI debugging

_Forwarding host TCP ports into the emulator so the Android app can hit `localhost:44300` and `localhost:44305` even though Android's localhost normally points at the emulator itself._

```bash
# Run from VS's adb prompt or any shell on PATH:
adb reverse tcp:44300 tcp:44300   # API host
adb reverse tcp:44305 tcp:44305   # auth server
# Verify:
adb reverse --list
```

**Key lines:** `adb reverse` is the inverse of `adb forward`: device-side TCP traffic is tunneled to the host. iOS Simulator does not need this — it shares the host's network stack. Remote iOS Simulator (Mac paired with a Windows VS host) cannot reach the Windows backend; backend must be hosted on the Mac in that scenario.

## Common mistakes

### Leaving `localhost` in Environment.ts (React Native) or appsettings.json (MAUI).

**Why wrong:** On Android emulators `localhost` resolves to the emulator itself, not the host. Physical devices have no route to the host's loopback interface. The auth flow either times out or returns 'connection refused'.

```csharp
Replace with the host's LAN IP (`ipconfig`/`ifconfig getifaddr en0`) or with `10.0.2.2` for the Android emulator. Update both `apiUrl` and `issuer` (RN) or `Authority` and `BaseUrl` (MAUI). On Android in tiered/microservice setups, prefer `adb reverse tcp:<port> tcp:<port>` so you can keep typing `localhost`.
```

### Forgetting to call `DisableTransportSecurityRequirement()` in PreConfigureServices.

**Why wrong:** OpenIddict refuses to issue tokens over HTTP by default. Mobile clients hitting the dev backend over HTTP get HTTP 400 with `error=invalid_request` and the body 'This server only accepts HTTPS requests.'

```csharp
Add the `#if DEBUG` block shown in the configuration_pattern section. Never deploy this to production — use a real cert (or the `yarn web` mkcert proxy for RN dev) instead.
```

### Relying on the .NET dev cert from a React Native or MAUI client.

**Why wrong:** React Native does not honour the .NET dev cert root, and Android won't trust user-installed certs at all without manifest changes. iOS Simulator behaves better but still complains.

```csharp
For React Native: use `yarn web` + mkcert (a locally-trusted CA) and reverse-proxy the backend, OR run the backend over plain HTTP in DEBUG. For MAUI Android: install the dev cert into the system trust store of the emulator AVD, or accept HTTP in DEBUG.
```

### Skipping the database migration before first backend run.

**Why wrong:** The OpenIddict DataSeedContributor seeds client_id/client_secret rows for `MyApp_App` (RN) and `MyApp_Maui`. Without those rows, the mobile login redirect returns `unauthorized_client`.

```csharp
Run the `*.DbMigrator` console project after creating the solution and before starting `*.HttpApi.Host`. The seeder is idempotent — safe to re-run.
```

### Removing `AbpAutofacModule` from the MAUI module's [DependsOn].

**Why wrong:** Several ABP framework features (interception, conventional registration of repositories, dynamic proxies for client services) are wired to Autofac. Removing it leads to runtime resolution failures or silently lost interceptors.

```csharp
Keep AbpAutofacModule. If you really need MS DI only, expect to manually rewire the affected pieces — see the linked support thread, but be aware this is unsupported.
```

### Using `App.Current.Properties` to store access/refresh tokens in MAUI.

**Why wrong:** Properties are saved as plain JSON in app-readable storage; access tokens leak to anyone with file-system access to the device.

```csharp
Use `Microsoft.Maui.Storage.SecureStorage.SetAsync(OidcConsts.AccessTokenKeyName, token)`. SecureStorage uses Keychain (iOS), KeyStore (Android), DPAPI (Windows).
```

### Confusing the React Native template with React JavaScript SPA.

**Why wrong:** ABP also has docs around React/Next.js for browser SPAs. The mobile template (`-m react-native`) is a different artefact: native binaries via Expo, not a browser bundle.

```csharp
If the goal is a web app, use Angular or MVC frontend; the React SPA option is documented separately and is not what `-m react-native` produces.
```

### Adding screens to the React Native app without registering them in a navigator.

**Why wrong:** `src/screens/MyScreen/` files won't be reachable until they are added to a stack/tab in `src/navigators/`. Reducers/sagas similarly need to be registered in the root store.

```csharp
Drop the screen file under `src/screens/MyScreen/` and add it to the appropriate navigator in `src/navigators/`. For state, add the reducer to the root reducer in `src/store/` and the saga to the root saga.
```

### Forgetting to declare the custom URL scheme for MAUI's redirect URI.

**Why wrong:** The OpenIddict redirect to `myapp://` must be claimed by the OS or the system browser cannot deep-link back into the app and the auth flow stalls at the browser tab.

```csharp
Android: add an `<intent-filter>` with `<data android:scheme="myapp" />` to the activity in `Platforms/Android/AndroidManifest.xml`. iOS: add `CFBundleURLTypes` with `CFBundleURLSchemes = ["myapp"]` to `Platforms/iOS/Info.plist`.
```

### Using the new ABP Studio template flow for MAUI without `--old`.

**Why wrong:** As of ABP 10.3 the MAUI startup template is not yet exposed through ABP Studio Templates. Running `abp new ... -t maui` without `--old` errors out or scaffolds an empty solution.

```bash
Use `abp new Acme.MyApp -t maui --old` (note the `--old` flag) until ABP Studio adds the MAUI template natively. The flag selects the legacy CLI/templating system.
```

## Version pins (ABP 10.3)

- ABP 10.3 React Native template requires Node.js >= 20.11. Earlier ABP versions allowed older Node.js — do not assume back-compat.
- MAUI startup template is commercial — needs a Team or higher license to scaffold.
- `abp new ... -t maui --old` is the correct command in 10.3; the `--old` requirement may disappear once MAUI is added to ABP Studio Templates. [uncertain — no roadmap commitment in 10.3 docs]
- ABP 6.0+ uses OpenIddict; older docs/community posts that reference IdentityServer4 do not apply. Both 10.3 mobile templates assume OpenIddict on the server.
- MAUI built-in `Settings` page features (theme, language, password, profile picture) are listed in the 10.3 docs — coverage of Mac Catalyst/Windows for these features is implied but not explicitly tested per the docs. [uncertain]
- Static C# Client Proxies are the documented MAUI approach in 10.3; dynamic proxies still work but are not what the template wires up.
- React Native template's `src/store/` historically uses Redux + Redux-Toolkit + Redux-Saga. ABP did not announce a migration to RTK Query in 10.3 docs. [uncertain — verify against the actual generated package.json before relying on saga]
- Volo.Abp.Maui.Client NuGet listing shows 10.0.2 publicly; the 10.3 package is consumed transitively by the template. Pin the version to match the rest of the ABP graph (do not mix 10.3 framework with 10.0 client).
- `DisableTransportSecurityRequirement()` is on `OpenIddictServerAspNetCoreBuilder` (via `UseAspNetCore()`), not on the OpenIddictBuilder root — easy to mis-call. API surface unchanged in 10.3 vs 10.x.
- iOS Simulator Remote (Mac paired to Windows host) cannot reach a Windows-hosted backend over the simulator's network — this is a Microsoft platform limit, not an ABP one, and is called out in the 10.3 MAUI doc.
